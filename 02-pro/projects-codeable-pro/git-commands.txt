git rebase -i HEAD~3
  para realizar un rebase interactive de los ultimos 3 commits (3 en el comando es para trabajar con los ultimos 3 commits)
    los rebase interactive pueden hacer varias cosas:
      - fixup (juntar commits y dejarlo con un unico mensaje)
      - editar especificos commits antiguos

para poder editar un commit antiguo (por ejemplo el commit trasantepenultimo)
  git rebase -i HEAD~3
    3 en este caso por si quieres editar el 3 commit desde el ultimo que hiciste
    dejas como edit al commit que necesites en la pantalla de seleccion
    cuando estes editando ese commit puedes hacer
      git reset --soft HEAD~1
        cambias lo que necesites
          ya luego al termina aplicas
            git rebase --continue

git push --force-with-lease
  para respetar commits de otras personas. te da warning

git checkout <nombre-rama>
git fetch origin
git reset --hard origin/<nombre-rama>

    para poder traer todos los cambios de remoto a tu rama <nombre-rama>
      y olvida todo lo que tengas en tu local (cuidado si tenias cambios en tu local)
    * user cuando hayan hecho git push --force-with-lease en tu rama

git reset --soft HEAD~1
  desempaqueta el ultimo commit y lo deja en staging area

git branch -m nombre-nuevo
  cambiar nombre de rama

    git push -u origin nombre-nuevo
      se sube una rama nueva. no se borra

        git push origin --delete nombre-viejo
          para borrar rama antigua

si se quiere editar un commit antiguo (agregando cambios o removiendo)
  git rebase -i HEAD~<N> (donde N es el numero de commit que se quiere editar)
  agregas los cambios necesarios
  git add .
  git commit --amend
  git rebase --continue

al resolver conflictos siempre con rebase, no con merge
  git checkout <nombre-rama-de-trabajo>
  git fetch origin
  git rebase origin/development
